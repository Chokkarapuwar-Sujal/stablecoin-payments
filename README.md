Blockchain payment system built with React, Node.js, and the Algorand blockchain for secure digital payments.

## Project Overview

This repository implements a blockchain-based digital payment system that enables users and merchants to send and receive payments using the Algorand blockchain.

The system supports:

- Wallet-to-wallet stablecoin transfers
- Merchant payment acceptance
- QR code-based payments
- Transaction monitoring
- On-chain settlement confirmation

For local demo mode, the project can also run as an ALGO payment system by setting the asset ID to `0`.

The project demonstrates how decentralized blockchain infrastructure can power secure, transparent, and fast digital payments.

## Problem Statement

Traditional payment systems rely on centralized intermediaries that introduce transaction fees, settlement delays, and limitations for cross-border payments.

Blockchain technology enables direct peer-to-peer payments without intermediaries while maintaining transparency and security.

Stablecoins combine the stability of traditional currencies with the programmability of blockchain technology.

This project demonstrates how stablecoin payments on Algorand can enable efficient and transparent digital payment systems.

## How Stablecoin Payments Work

1. User enters a receiver wallet address or scans a merchant QR code.
2. User can either connect Pera Wallet or use backend mnemonic mode.
3. A transaction is created using the Algorand SDK.
4. The transaction is submitted to the Algorand blockchain.
5. Backend monitors the transaction until confirmation.
6. Settlement confirmation is displayed to the user.

## Installation Guide

### 1. Prerequisites

- Node.js 18+
- Algorand Testnet Access
- Pera Wallet

### 2. Configure Environment

Copy the environment template:

```bash
cp .env.example .env
```

PowerShell (Windows):

```powershell
Copy-Item .env.example .env
```

Example `.env` configuration:

```env
PORT=4000
ALGOD_SERVER=https://testnet-api.algonode.cloud
ALGOD_PORT=
ALGOD_TOKEN=
INDEXER_SERVER=https://testnet-idx.algonode.cloud
INDEXER_PORT=
INDEXER_TOKEN=
STABLECOIN_ASSET_ID=0
SENDER_MNEMONIC=

VITE_API_BASE_URL=http://localhost:4000/api
VITE_ALGOD_SERVER=https://testnet-api.algonode.cloud
VITE_ALGOD_PORT=
VITE_ALGOD_TOKEN=
VITE_STABLECOIN_ASSET_ID=0
```

Use `STABLECOIN_ASSET_ID=0` and `VITE_STABLECOIN_ASSET_ID=0` for ALGO demo mode.
Replace both with the same ASA ID for real stablecoin mode.

## Demo Workflow

1. Start the project:

```bash
npm run dev:all
```

2. Generate a wallet:

```bash
node setupWallet.js
```

3. Fund the generated wallet using the Algorand TestNet faucet.
4. Open the app at `http://localhost:5173`.
5. Send a payment from the `Send Payment` page.
6. Monitor transaction updates in the `Transaction Status` page.

This helps reviewers and recruiters test the project quickly with minimal setup.

## Demo Mode (ALGO Transfer)

For quick testing, the system can operate in ALGO transfer mode instead of stablecoin ASA transfer.

Set the following values in `.env`:

```env
STABLECOIN_ASSET_ID=0
VITE_STABLECOIN_ASSET_ID=0
```

This allows sending ALGO on Algorand TestNet for demonstration purposes without deploying a custom stablecoin asset.

### One-command startup (optional)

From project root:

```bash
npm install

```

This starts backend and frontend together.

### Backend mnemonic mode (simplest testing)

You can test without Pera Wallet by letting backend sign transactions with `SENDER_MNEMONIC`.

1. Generate a wallet:

```bash
node createWallet.js
```

2. Copy output mnemonic into `.env`:

```env
SENDER_MNEMONIC=your 25-word mnemonic here
```

3. Fund generated address from Algorand TestNet faucet:

- `https://dispenser.testnet.aws.algodev.network/`

4. Set both asset IDs:

```env
STABLECOIN_ASSET_ID=0
VITE_STABLECOIN_ASSET_ID=0
```

Use `0` for ALGO demo mode. Use a real ASA ID only if you want stablecoin token transfers.

In this mode, wallet connection is optional in UI.

### Setup script (recommended)

Use the setup script to auto-generate a wallet and write `SENDER_MNEMONIC` into `.env`.

```bash
node setupWallet.js
```

The script prints wallet address and mnemonic, then updates `.env`.

For quick demo mode (ALGO transfer), keep:

```env
STABLECOIN_ASSET_ID=0
VITE_STABLECOIN_ASSET_ID=0
```

`0` means ALGO transfer instead of ASA stablecoin transfer.

Security rules:

- Never commit `.env`.
- Commit only `.env.example`.
- If mnemonic is exposed, rotate wallet and move funds.
- If transaction status seems stuck on `pending`, wait a few seconds and retry because public Algod/Indexer services can lag.

## Example Usage

1. Connect Pera Wallet
2. Enter receiver wallet address
3. Enter payment amount
4. Send payment

### Backend Mnemonic Payment

1. Leave wallet disconnected
2. Ensure `SENDER_MNEMONIC` is set in `.env`
3. Enter receiver wallet address
4. Enter payment amount
5. Backend signs and submits the transaction

### Merchant QR Payment

1. Merchant generates QR payment code
2. Customer scans QR code
3. Payment transaction is submitted
4. Blockchain confirms settlement

## API Documentation

### Send Payment

- `POST /api/payments/send`
- Send stablecoin payment.

### Generate Merchant QR

- `POST /api/payments/qr`
- Generate merchant payment QR code.

### Check Transaction Status

- `GET /api/payments/status/:txId`
- Returns blockchain confirmation status.

## Transaction Monitoring

After submitting a payment:

1. Transaction ID is generated.
2. Backend queries the Algorand blockchain.
3. Confirmation status is retrieved.
4. The frontend displays payment confirmation.
